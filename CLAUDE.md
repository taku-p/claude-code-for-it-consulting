# プロジェクトルール

## クリップボードへの自動コピー（最重要ルール）

ユーザーがコピペして貼り付けそうなテキストを生成したら、**必ず自動でクリップボードにコピーすること**。
ユーザーに「コピーして」と言われる前に、先回りしてコピーしておく。

以下のような場面では必ずコピーする：
- ツイート、返信文、SNS投稿文を生成したとき
- メール文面を生成したとき
- どこかに貼り付ける前提のテキストを生成したとき
- ユーザーが「作って」「書いて」と依頼した短いテキスト

### コピー方法（絶対に守ること）

**`clip.exe` や `clip` コマンドは絶対に使わないこと。日本語が文字化けする。**

必ず以下のファイル経由の方法を使う：

```bash
# 1. まずテキストをファイルに書き出す（Write ツールを使用）
# 2. clip_utf8.ps1 でクリップボードにコピー
powershell.exe -ExecutionPolicy Bypass -File "clip_utf8.ps1" -File "コピーしたいファイルのパス"
```

clip_utf8.ps1 は [System.IO.File]::ReadAllText で UTF-8 として読み取り、Set-Clipboard でコピーする。
bash のパイプ経由だとエンコーディングが壊れるため、**必ずファイル経由**で行うこと。

### フロー

1. テキストを生成する
2. ファイルに UTF-8 で書き出す（Write ツール）
3. clip_utf8.ps1 でクリップボードにコピー
4. ユーザーに「クリップボードにコピーしました。貼り付けてください」と伝える

### 一時ファイルのパスに注意

`/tmp/` はWindows上のPowerShellから見えないことがある。一時ファイルはプロジェクトディレクトリ内に作ること。

## X（Twitter）記事の作成ルール（絶対に守ること）

### プレーンテキストは使わない

Xの記事エディタに貼り付ける記事は、**必ずHTMLで作る**。
プレーンテキスト（Unicode太字変換）ではXの記事エディタで見出し・太字が効かない。

### 正しい方法

1. **HTMLファイルを作成する**（`<h2>`で見出し、`<b>`で太字）
2. **ブラウザで開く**（`start "" "ファイル名.html"`）
3. **ユーザーがCtrl+A → Ctrl+C → Xの記事エディタにCtrl+V**
4. これでXが見出し・太字を自動認識する

参考テンプレート: `x_article_rich.html`, `x_article_ads_rich.html`

### やってはいけないこと

- unicode_convert.py でプレーンテキストに変換してXの記事に貼る → **見出しが効かない、太字が見えない**
- unicode_convert.py はX記事以外の用途（ツイート本文等）にのみ使う

## カバー画像の作成

Xの記事にはカバー画像（5:2、1200×480px）を設定する。

1. HTMLでデザインを作成（参考: `cover.html`, `cover_ads.html`）
2. Puppeteerでスクショを撮る（参考: `capture.js`, `capture_ads.js`）
3. `deviceScaleFactor: 2` + `--force-color-profile=srgb` を忘れない
