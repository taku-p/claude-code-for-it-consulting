# Claude Codeを触ってみたら、ITコンサル業務にかなり使えそうだった話

> 最近話題のClaude Codeが気になって、自腹で約3万円（$200プラン×1ヶ月）課金して触ってみました。
> 結論、「これ業務に導入したら相当生産性上がるのでは？」と感じたので、試したことをまとめます。

---

## はじめに

普段ITコンサルとして働いていて、Miroでフロー図を書いたり、Excelで設計書を作ったり、打ち合わせのあとにタスクを整理したり...という業務が日常的にあります。

「AIツールで何か効率化できないかな」と思って、自腹でClaude Codeに課金して試してみました（Max 5xプラン $200/月 ≒ 約3万円）。

GeminiやNotebookLMは使ったことがあったんですが、Claude Codeはちょっと毛色が違っていて、**ローカルのファイルを直接読み書きできる**のと、**APIスクリプトをその場で書いて実行してくれる**のが特徴です。

---

## Claude Codeって何？

Anthropic社のCLI（ターミナルで動く）AIアシスタント。ざっくり言うと：

- PCの中のファイル（Excel、PDF、JSON等）を直接読み書きできる
- 「こういうスクリプト書いて」と言うとPythonとか書いて実行してくれる
- MCP（Model Context Protocol）という仕組みで、MiroやSlack等の外部ツールとも連携可能
- 過去のやり取りを覚えていて、前回の続きから作業できる

---

## 試してみたこと

### 1. MiroのボードをAPI経由で自動操作してみた

コンサルの仕事でMiroはかなり使うんですが、フロー図の更新って地味に時間かかりますよね。付箋を追加して、コネクタ繋ぎ直して、位置調整して...。

試しに「MiroのAPIでボードの中身を取得して」と頼んでみたら、PythonスクリプトでMiro APIを叩いて、ボード上の全アイテム（付箋・コネクタ・座標）をJSON形式で取得してくれました。

さらに「このフローに新しいステップを追加して」と言ったら、既存のフロー構造を読み取った上で、新しい付箋とコネクタを追加するスクリプトまで生成。APIのエラー（アスペクト比制限とか）が出ても、エラー内容を読んで自分で修正してくれました。

**これは正直驚きました。** 手作業で2〜3時間かかるフロー更新が、30分くらいで終わる感覚です。コネクタの張り替え漏れもないし、座標計算も正確。

---

### 2. ExcelのテンプレートからExcel定義書を自動生成してみた

試しに、既存のExcel設計書テンプレートを読み込ませて、「このフォーマットに合わせて別の定義書を作って」とお願いしてみました。

すると：

1. テンプレートの列構成・書式を解析
2. 要件をYAML形式で整理
3. YAMLからExcelを生成するPythonスクリプトを作成
4. Excel出力

という流れを全部やってくれました。50項目以上ある定義書でも、テンプレート通りのフォーマットで出力されます。

項目の追加・変更があっても、YAMLを直してスクリプトを再実行するだけなので、**手戻りが発生しても楽**です。

---

### 3. 大量のExcel/PDFを一気に読み解かせてみた

プロジェクトに入ると、過去のドキュメントが大量にありますよね。Excel仕様書、PDF設計書、画面キャプチャ...。

Claude Codeに複数ファイルをまとめて渡して「これらの内容を整理して」と頼むと：

- 各ファイルの構造と内容を読み取り
- ドキュメント間の関連性を整理
- 項目名の揺れや不整合を検出
- Markdownで横断的なサマリーを出力

キャッチアップの時間が大幅に減りそうです。「ドキュメントを読む」作業をAIに任せて、自分は**判断に集中**できるイメージ。

---

### 4. As-Is/To-Beフロー設計書のドラフトを作ってみた

既存のフロー文書を読ませた上で、「ここに新しいプロセスを追加したTo-Beフローを作って」と伝えたら、As-Isとの差分が分かるようにTo-Beフロー設計書のドラフトを生成してくれました。

もちろん最終的には人間がレビューして調整するんですが、**ゼロからドラフトを書く時間がほぼなくなる**のは大きい。口頭で説明したレベルの情報から、それなりに形になった設計書が出てきます。

---

### 5. 打ち合わせメモからタスクを自動抽出してみた

上司との打ち合わせ後、メモからタスクを整理する作業。地味だけど毎回やる作業です。

打ち合わせのメモをそのままClaude Codeに渡して「タスクを整理して」と言うだけで：

- タスクの抽出・分類
- 優先度・依存関係の整理
- 報告用メッセージの整形

まで自動でやってくれます。30分かかってた作業が5分。

---

### 6. API連携スクリプトが自然言語で書ける

これが一番汎用性が高いかもしれません。

「MiroのAPIで〇〇して」「SlackのAPIで〇〇して」のように日本語で指示するだけで、APIドキュメントの知識がなくてもスクリプトが生成・実行されます。

APIのエラーが返ってきても、Claude Codeがエラー内容を読んで原因を特定し、修正版を出してくれるので、**トライ&エラーのサイクルが異常に速い**です。

## GeminiやNotebookLMとどう違うの？

社内でGeminiやNotebookLMを使っている人もいると思うので、比較してみます。

| 観点 | Gemini / NotebookLM | Claude Code |
|------|----------------------|-------------|
| ファイルアクセス | アップロード or Google Drive連携 | ローカルファイルに直接アクセス |
| Excel操作 | 閲覧・要約は可能、生成は不可 | 読み取り＋スクリプトでExcel生成まで可能 |
| スクリプト実行 | 不可 | その場でPython等を生成・実行 |
| 外部ツール連携 | Google系に限定 | MCP経由でMiro、Slack等と連携可能 |
| 作業の継続性 | セッション単位 | プロジェクト単位で文脈を保持 |
| 得意なこと | ドキュメントの要約・Q&A | ファイル操作・API連携・自動化 |

GeminiやNotebookLMは「ドキュメントを読んで質問に答える」のが得意です。一方、Claude Codeは**「ファイルを読んで、スクリプトを書いて、実行して、結果を出す」**まで一気通貫でやってくれます。

例えるなら、Geminiが「優秀なリサーチャー」だとすると、Claude Codeは**「手を動かしてくれるアシスタント」**です。用途が違うので、併用するのもアリだと思います。

## 導入するとしたら気をつけること

1. **CLIベースなので、最低限のターミナル操作は必要**
   - ただし、Claude Code自身がコマンドを実行してくれるので、実質的なハードルは低い

2. **機密情報の取り扱いルールは必要**
   - ファイルの内容はAnthropicのAPIに送信される
   - 社内のセキュリティポリシーに合わせた運用ガイドラインの策定が必要

3. **外部API連携にはAPIトークンの発行が必要**
   - MiroやSlack等との連携には、各サービスのAPIキーが必要（初回のみ）

## まとめ

正直、触る前は「エンジニア向けのツールでしょ？」と思っていました。

でも実際に試してみると、**Miroのフロー図更新、Excel設計書の生成、ドキュメントの読み解き**など、ITコンサルの日常業務にかなり使えることが分かりました。

特にMiro APIとの連携は、フロー図の作成・更新が多いコンサルワークにおいて、かなりインパクトがありそうです。

自腹で約3万円使ってみた結論として、会社で使うなら月$100（≒15,000円）のMaxプランで十分だと思いますし、月20〜40時間の効率化が見込めるなら、十分すぎるほど元が取れるツールだと感じました。会社として導入すれば、チーム全体の生産性向上につながるポテンシャルがあると思います。
