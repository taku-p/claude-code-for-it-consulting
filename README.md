- 社内でのAIツール導入を検討中の方
- Claude Codeの具体的な活用イメージが欲しい方

---

## Claude Codeとは

Anthropic社が提供するCLI（コマンドライン）ベースのAIアシスタントです。
ChatGPTのようなチャット型ではなく、**ローカルのファイルシステムに直接アクセス**できるため、以下のような特徴があります。

| 特徴 | 詳細 |
|------|------|
| ファイル操作 | ローカルのExcel、PDF、YAML、JSON等を直接読み書きできる |
| API連携 | 外部サービスのAPIを叩くスクリプトをその場で生成・実行 |
| Git連携 | コードのコミット、PR作成、差分確認を自然言語で指示可能 |
| MCP対応 | Model Context Protocolで外部ツール（Miro、Salesforce等）と直接連携 |
| セッション記憶 | 過去の作業内容を記憶し、文脈を保持した継続作業が可能 |

## 実務で活用した具体的ユースケース

### 1. Miro APIと連携した業務フローの自動操作

**課題：** 業務フロー図（Miro）の更新作業が手作業で時間がかかる。フロー変更のたびに付箋の追加・コネクタの張り替え・レイアウト調整を人力で行っていた。

**Claude Codeでやったこと：**

1. Miro APIを使って既存ボードの全アイテム（付箋・コネクタ・座標）をJSON形式で取得
2. ボードの現在状態をスナップショットとして保存し、構造を可視化
3. 新しいフロー（例: 電子署名プロセスの追加）を設計し、Miro APIで自動反映するPythonスクリプトを生成
4. APIの制約（アスペクト比制限等）でエラーが出た場合も、Claude Codeがエラーを読み取り即座に修正

```
実行例:
「Miroの既存フローにCloudSignの電子署名ステップを追加して」
→ ボード読み取り → フロー設計 → Pythonスクリプト生成 → API実行 → エラー修正 → 完了
所要時間: 約30分（手作業なら2〜3時間）
```

**効果：** フロー図の更新が5〜6倍速くなった。特に「既存のフローを壊さずに新しいステップを挿入する」作業で、コネクタの張り替え漏れがなくなった。

---

### 2. Salesforceオブジェクト定義書の自動生成

**課題：** Salesforceのカスタムオブジェクト定義書をExcelで作成する作業。50以上のカスタムフィールドの定義（API名、型、選択リスト値、プロファイル別権限等）を既存のExcelテンプレートに合わせて記載する必要があった。

**Claude Codeでやったこと：**

1. 既存の定義書テンプレート（Excel）を読み取り、フォーマットを解析
2. 要件をYAMLで構造化して管理（人間が読みやすく、差分管理も容易）
3. YAMLからExcel定義書を自動生成するPythonスクリプトを作成
4. 既存の参照ドキュメント（別のオブジェクト定義書）と項目の整合性をチェック

```
実行例:
「この既存の定義書テンプレートに合わせて、見積オブジェクトの定義書を作って」
→ テンプレート解析 → YAML定義 → 51フィールド定義 → Excel生成
所要時間: 約2時間（手作業なら丸1日）
```

**効果：** 定義書1本あたり6〜8時間の作業が2時間に短縮。フィールドの追加・変更時もYAMLを編集して再生成するだけ。

---

### 3. 既存ドキュメント（Excel/PDF）の読み取りと構造化

**課題：** プロジェクトに参画すると、大量のExcel仕様書やPDFの設計書が既に存在している。これらを読み解いて現状を把握するのに時間がかかる。

**Claude Codeでやったこと：**

1. 複数のExcelファイル（オブジェクト定義書、請求マスターシート等）を一括読み取り
2. PDFの画面レイアウト資料を読み取り、フィールドの対応関係を把握
3. 読み取った情報をMarkdownやYAMLで構造化し、横断的に整理
4. 既存ドキュメント間の不整合（項目名の揺れ、型の不一致等）を検出

```
実行例:
「この3つのExcelファイルを読んで、見積オブジェクトに必要な項目を整理して」
→ Excel読み取り → 項目抽出 → 重複・不整合チェック → 統合定義YAML出力
```

**効果：** 新規参画時のキャッチアップが圧倒的に速くなる。「ドキュメントを読む」作業自体をAIに任せ、自分は「判断」に集中できる。

---

### 4. 業務フロー設計書の生成

**課題：** To-Be業務フローの設計書を、既存のAs-Isフローとの差分が分かるように作成する必要があった。

**Claude Codeでやったこと：**

1. 既存のAs-Isフロー文書（Markdown）を読み取り
2. 新規要件（電子署名プロセス、CloudSign連携等）を口頭で説明
3. As-Isとの差分を明確にしたTo-Beフロー設計書をMarkdownで生成
4. 生成した設計書の内容をMiroのフロー図に反映（上述のMiro API連携）

**効果：** 設計書のドラフト作成が口頭説明レベルの情報から可能になった。「考える」→「書く」のギャップが大幅に縮まった。

---

### 5. 打ち合わせ後のタスク整理・構造化

**課題：** 上司との打ち合わせ後、議事メモからタスクを抽出・優先度付けする作業。

**Claude Codeでやったこと：**

1. 打ち合わせのメモ（テキスト）を投げるだけで、タスクを自動抽出
2. 各タスクに対して、次のアクション・依存関係・優先度を整理
3. 上司への報告用に整形したメッセージを生成
4. フロー図の修正指示があれば、そのままMiro API連携で実行

```
実行例:
「この議事メモからタスクを整理して、上司に共有できる形にして」
→ メモ読み取り → タスク抽出 → 優先度付け → 報告メッセージ生成
所要時間: 5分（手作業なら30分）
```

---

### 6. APIスクリプトの即時生成と実行

**課題：** 外部サービスのAPI（Miro、Salesforce等）を使った自動化をしたいが、APIドキュメントを読んでスクリプトを書く時間がない。

**Claude Codeでやったこと：**

- MiroのREST APIを使ったボード操作スクリプト（Python）を自然言語指示で生成
- Salesforce CLIと連携したメタデータ取得
- APIのエラーレスポンスをClaude Codeに渡すと、原因特定と修正を自動で実行

**効果：** APIドキュメントを読む時間がほぼゼロに。「やりたいこと」を日本語で伝えるだけでスクリプトが生成される。

## ChatGPTとの違い

| 観点 | ChatGPT | Claude Code |
|------|---------|-------------|
| ファイルアクセス | アップロードが必要 | ローカルファイルに直接アクセス |
| Excel読み書き | 限定的 | 読み取り＋スクリプトで生成可能 |
| API実行 | 不可 | その場でスクリプト生成・実行 |
| Git連携 | 不可 | コミット、PR作成、差分確認 |
| 外部ツール連携 | プラグイン（限定的） | MCP経由でMiro、Salesforce等と直接連携 |
| 作業の継続性 | セッションごとにリセット | プロジェクト単位で文脈を保持 |
| セキュリティ | クラウドにデータ送信 | ローカル実行（機密ファイルをアップロード不要） |

---

## 導入時の注意点

1. **CLIベースのため、最低限のターミナル操作知識が必要**
   → ただし、Claude Code自身がコマンドを実行してくれるため、実質的なハードルは低い

2. **機密情報の取り扱い**
   → ファイルの内容はAnthropicのAPIに送信されるため、社内規定に沿った運用ルールの策定が必要

3. **API連携にはトークンの取得が必要**
   → Miro、Salesforce等の連携には各サービスのAPIトークンが必要。初回セットアップのみ

## まとめ

Claude Codeは「コードを書くためのツール」ではなく、**ITコンサル業務の生産性を根本から変えるツール**です。

特に以下のような業務との相性が非常に良いです：

- **Miro等のSaaSとAPI連携した業務自動化**
- **Salesforce等のシステム設計ドキュメント作成**
- **既存ドキュメントの読み解き・構造化**
- **業務フローの設計・可視化**

月額3,000〜15,000円の投資で、月30時間以上の作業時間を削減できる実績があります。
特にMiro APIとの連携は、フロー図の作成・更新が多いコンサル業務において大きなインパクトがあります。
