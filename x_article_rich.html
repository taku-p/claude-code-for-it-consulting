Claude Codeに3万円自腹で課金したITコンサルが、若手の仕事がなくなると確信した話

はじめに

普段ITコンサルとして働いてる。Miroでフロー図書いて、Excelで設計書作って、打ち合わせ後にタスク整理して...っていう、まあよくある日常。

最近Claude Codeっていうツールが気になって、自腹で約3万円（$200プラン×1ヶ月）課金して触ってみた。

結論から言うと、若手コンサルがやってる作業の大半は、これで代替できる。

「いやいや、大げさでしょ」って思った人、最後まで読んでほしい。

そもそもClaude Codeって何？

Anthropic社が出してるCLI（ターミナルで動く）のAIアシスタント。

普通のAIチャットとの決定的な違いは、PC内のファイルを直接読み書きできることと、スクリプトをその場で書いて実行してくれること。

つまり「相談相手」じゃなくて「手を動かしてくれるアシスタント」。

やってみたこと① Miroのフロー図を自動操作

コンサルの仕事でMiroは毎日使う。で、フロー図の更新ってめちゃくちゃ地味に時間かかる。付箋追加して、コネクタ繋ぎ直して、位置調整して...。

Claude Codeに「このフローに電子署名のステップを追加して」って言ったら、

1. Miro APIでボードの全アイテムを取得
2. 既存フロー構造を読み取り
3. 新しいステップを追加するPythonスクリプトを生成
4. API実行→エラー出たら自分で修正→完了

手作業2〜3時間が30分に。

コネクタの張り替え漏れもゼロ。正直、ここが一番衝撃だった。

やってみたこと② Excel定義書の自動生成

既存のExcelテンプレートを読み込ませて「このフォーマットで定義書作って」と頼んだ。

テンプレートの列構成を解析→要件をYAMLで整理→Excelを自動生成。50項目以上の定義書がテンプレート通りに出てくる。

修正が入っても、YAML直して再生成するだけ。

丸1日かかる作業が2時間に。

やってみたこと③ 大量のドキュメントを一気に読み解き

プロジェクトに入ると、前任者が作った大量のExcel仕様書やPDF設計書がある。これ読むだけで1〜2日かかるやつ。

Claude Codeに全部渡して「内容を整理して」と頼んだら、

・各ドキュメントの構造を解析
・ドキュメント間の不整合を検出（項目名の揺れとか）
・Markdownで横断サマリーを出力

「ドキュメントを読む」仕事がほぼなくなる。自分は判断に集中できる。

やってみたこと④ 議事録からタスク自動抽出

地味だけど毎回やるやつ。打ち合わせメモからのタスク整理。

メモをそのまま渡すだけで、タスク抽出→優先度付け→報告用メッセージ整形まで自動。

30分が5分に。

やってみたこと⑤ API連携が日本語で書ける

「MiroのAPIで〇〇して」って日本語で言うだけでスクリプトが出てくる。APIドキュメント読む必要がない。

エラーが出ても、Claude Codeが原因特定して修正版を出してくれる。トライ&エラーが異常に速い。

GeminiやNotebookLMと何が違う？

ここ大事。「Geminiでよくない？」って思う人多いと思う。

Gemini / NotebookLM → ドキュメントを読んで質問に答える
Claude Code → ファイルを読んで、スクリプト書いて、実行して、結果を出す

Geminiが「優秀なリサーチャー」なら、Claude Codeは「手を動かすアシスタント」。

ドキュメントの要約とかはGeminiで十分。でも、Excel生成したりMiro操作したり、実際にアウトプットを作るのはClaude Codeじゃないとできない。

若手コンサルへ

ここまで読んで気づいたと思うけど、ITコンサルの若手が日常的にやってる作業、かなりの部分がAIで代替可能になってきてる。

・フロー図の作成・更新
・設計書のドラフト
・既存ドキュメントの読み解き
・議事録整理・タスク抽出

これ、若手の仕事の大半じゃない？

ただ、逆に言えばこういうツールを使いこなせる人は、同じ時間で3〜5倍のアウトプットが出せる。代替される側になるか、使いこなす側になるか。

アクセンチュアとかの大手は既にAIツールの導入がどんどん進んでる。中堅以下のファームこそ、早めに検討した方がいい。

月$100（≒15,000円）/人の投資で、月20〜40時間の作業を効率化できるポテンシャルがある。別に全員に入れる必要はない。まず1人、2人に使わせてみるだけでも効果は分かる。

自腹で3万円出して試した人間が言うから、少なくとも「試す価値はある」のは間違いない。
