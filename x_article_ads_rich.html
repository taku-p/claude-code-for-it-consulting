広告運用、コードで自動化したら世界変わった話

はじめに

広告運用って「管理画面を毎日ポチポチする仕事」だと思ってた。

クリエイティブはデザイナーに依頼して、配信状況は管理画面で確認して、レポートはスプレッドシートにまとめて...。

これを全部コードで自動化したら、「異常が起きた時だけ対応する仕事」に変わった。やったことを共有する。

クリエイティブの自動生成

広告クリエイティブ、デザイナーに毎回依頼してたらA/Bテストなんて回せない。1パターン作るのに数日かかるのに、テストしたいバリエーションは10個以上ある。

解決策：Next.jsでテンプレートを作って、Puppeteerでスクショを撮る。

仕組みはシンプルで、

・テンプレート（デザインパターン）をHTML/CSSで3種類作る
・写真×コピー×テンプレートの組み合わせをデータとして定義
・Puppeteerが1枚ずつページを開いてスクリーンショットを撮る
・1コマンドで全バリエーションが画像として書き出される

これで「テンプレート3種 × 写真3枚 × コピー3パターン」みたいな組み合わせを一撃で生成できる。

デザイナーが作った元デザインをコードに落とし込む初期コストはあるけど、2回目以降は写真とコピーを差し替えるだけ。

ハマったポイント

地味だけど重要なやつ。

・リモートフォント（Google Fonts等）はヘッドレスブラウザでCORSエラーになる → ローカルにフォントファイルを置いて解決
・ヘッドレスモードだと色が薄くなる → sRGBカラープロファイルを強制指定
・Next.jsの開発UIがスクショに映り込む → DOM操作で非表示にしてからキャプチャ

これ知らないと「なんか色が違う」「謎のUIが映ってる」で数時間溶ける。

配信管理の自動化

Meta広告の配信管理、管理画面を毎日開いてチェックするの面倒すぎる。特に複数キャンペーンあると見落としが怖い。

解決策：GitHub Actionsで30分おきにMeta APIを叩いて、異常があればLINEに通知。

やってること：

・30分ごとに全広告の配信ステータスを取得
・前回の状態と比較して、変化があればLINE通知（審査落ち、配信停止など）
・CPC・CTR・フリークエンシーに閾値を設定、超えたらアラート
・毎日19時に日次レポートを自動送信
・毎週月曜に週次サマリーを自動送信
・学習期間終了後に最適化レポートを自動生成

閾値の設定例

自分の場合の目安：

・CPC ¥80超え → 危険、要対応
・CPC ¥50超え → 警告
・CTR 0.5%以下 → 危険
・フリークエンシー 3.0超え → クリエイティブ疲弊

最低インプレッション500〜1,000くらいは溜まってからじゃないと判定が不安定になるので注意。

レポートの自動化

広告主への報告も全部自動化した。

・日次：今日の消化金額、クリック数、CTR、CPCを広告セットごとに集計
・週次：先週比の推移、CPC改善/悪化の判定
・最適化：広告セットをCPCでランキング、予算の寄せ先を提案

全部LINEに飛ぶから、管理画面を開かなくても状況がわかる。クライアントとのLINEグループに直接飛ばせば、レポート作成の手間もゼロ。

使ってる技術

・Next.js + TypeScript + Tailwind CSS（クリエイティブのテンプレート）
・Puppeteer（ヘッドレスブラウザで画像書き出し）
・Meta Marketing API v22.0（配信データ取得・操作）
・LINE Messaging API（通知・レポート送信）
・GitHub Actions（30分おきのスケジュール実行）
・Node.js（全体の接着剤）

やってみての感想

手作業でやってた時は、広告運用＝管理画面を毎日ポチポチする仕事だった。自動化したら、異常が起きた時だけ対応する仕事に変わった。

クリエイティブも、最初のテンプレート構築さえ終われば「写真とコピーをデータに追加して1コマンド」で量産できる。A/Bテストの回転が圧倒的に速くなった。

広告運用をコードで管理するメリットは、再現性。同じ仕組みを別の案件にそのまま横展開できる。クライアントが増えても工数が線形に増えない。

全部Claude Codeと一緒に作った。
